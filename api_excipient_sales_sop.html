Pharma B2B Sales SOP｜API・Excipient・CDMO

API / Excipient 客戶應對 SOP

適合剛進入醫藥原料代理、API、賦形劑、CDMO 技術型業務的新手使用。 核心目標不是馬上知道所有答案，而是建立可信任、可追蹤、可回覆的專業流程。

核心原則：先聽，再問，再答。客戶要的是可靠窗口，不是百科全書。

Step 1｜先判斷客戶角色

客戶角色 常問問題 真正關心 你的應對重點
RD 研發 原料特性、應用、替代品、配方建議 能不能做得出來 先問劑型、用途、目前困難
QA / QC GMP、CoA、規格、稽核、變更管理 品質與供應是否穩定 確認文件版本與原廠品質資料
RA 法規 DMF、CEP、LOA、ASMF、註冊支援 能不能申請藥證 確認申請市場、文件可提供程度
採購 價格、MOQ、交期、付款條件 能不能買、成本是否合理 確認需求量、時程、是否已通過內部評估
CDMO / PM 技術能力、專案時程、批量、製程支援 專案能不能順利執行 釐清階段、規模、技術缺口

Step 2｜先問 3 個問題

目前是哪個階段？

新產品開發、第二供應商評估、量產供貨、法規申請、還是品質問題處理？

請問實際用途是什麼？

錠劑、膠囊、ODT、注射劑、外用、保健食品，或僅是研發測試？

目前最大的困擾是什麼？

價格、缺貨、法規文件、技術問題、品質問題、供應商替代，或時程壓力？

Step 3｜問題分類與回應方式

類型 例子 標準回應
文件類 DMF、CEP、GMP、CoA、SDS 我確認最新版本後提供給您。
規格類 粒徑、水分、Assay、殘留溶劑 我查閱最新規格書後回覆您。
技術類 壓錠性、溶解度、相容性、賦形劑選擇 我先了解應用情境，再與原廠確認。
商務類 價格、MOQ、Lead time、樣品 我確認庫存與交期後提供報價資訊。

不知道時怎麼說

「這部分我想確認一下最新資料，再回覆您。」
「我不希望提供不完整資訊，我先與原廠確認。」
「不同應用條件可能會影響結果，我先了解您的情況。」
「我今天或明天整理後回覆您。」

Step 5｜24 小時回覆原則

即使還沒有完整答案，也要回覆進度，避免客戶覺得你消失。

「目前已向原廠確認，預計明天下午提供完整資料。」
重點：不知道不是問題；沒有追蹤、沒有回覆，才會失去信任。

完整處理流程

接收問題 → 確認角色 → 釐清階段與用途 → 分類問題 → 查文件 / 找原廠 → 回覆進度 → 整理成 FAQ

拜訪後 30 分鐘內記錄

客戶產品線 真正決策者 目前困難 需要文件 下一步 下次聯絡時間
